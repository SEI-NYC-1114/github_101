6. What is the purpose of a .gitignore file?

7. What is a fork? Why would you want to have a fork?
A fork is your local, independent copy of the original remote repository. You would want a fork to avoid changing the original repository directly and risk affecting it. 

8. What is a clone? How many clones can you have?
Similar to a fork, a clone is a copy of the original Git repository. However, a clone links to the original repo and is not independent. 
You can have more than one clone, but this is typically not the case.

7. What is a fork? Why would you want to have a fork?
a: forking a repo creates a copy of it. You can then clone that repo to your local machine and make changes before commiting and pushing them. This allows multiple developers to work together 
8. What is a clone? How many clones can you have?
a: a clone is a local repo of the github repo you forked. You can have unlimited clones.
